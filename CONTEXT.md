# まず覚える (mazuoboeru)

学んだことをクイズ化して反復で覚える、マルチユーザーの公開学習サービス。
クイズ本体は必ず公開、個人の学習履歴・PII は非公開。

## Language

**Quiz (クイズ)**:
1人のユーザ（作者）が作って公開する、設問の集合体。タイトル、説明、複数の [[Question]] を含む。作成と同時に自動公開される（非公開オプションは存在しない）。
_Avoid_: テスト、試験、問題集

**Session (セッション)**:
ブラウザで Web UI を使う人間ユーザの認証状態。Cookie ベース（HttpOnly + Secure + SameSite）。サーバ側で `session` テーブル管理。
_Avoid_: ログイン情報、認証クッキー

**PAT (Personal Access Token)**:
CLI または AI エージェント（Claude 等）が API を叩くための Bearer トークン。Web の設定画面でユーザが自分で発行し、平文表示は発行直後のみ。失効・スコープを持つ。クイズの量産（AI 駆動の作成）が主な用途。
_Avoid_: API キー、シークレット、アクセスキー、APIトークン（曖昧）

**Publish (公開する)**:
作者（または作者の [[PAT]] を使う委任エージェント）による **意図的・不可逆** な明示操作。`draft` → `published` の一方向で、一度公開したクイズを `draft` に戻すことはできない（[[0002-publish-flow-and-edit-rules]]）。チャネルは問わない（Web の「公開する」ボタン、または PAT 経由の明示的な publish 呼び出し）が、いずれも「明示的な1ステップ」である点は共通（暗黙の副作用としては起こらない）。
_Avoid_: リリース、出題、公表（曖昧）

**Draft (下書き)**:
作者本人のみが見える、作成途中のクイズ。タイトルや設問が未完成でも保存できる。`Publish` の対義語的状態。
_Avoid_: 非公開、プライベート（mazuoboeru には private 状態は存在しない）

**Hidden (非表示)**:
モデレータが違反コンテンツに対して付与する状態。一般ユーザからは検索/直リンク共に閲覧不可（403）。作者本人だけ自身の hidden 状態を確認できる。`Draft` とは別概念（draft は作者の意思、hidden は他者の介入）。
_Avoid_: ブロック、削除、停止（曖昧）

**Attempt (挑戦)**:
1人のユーザが1つのクイズに挑む1セッション分の記録。`attempt_answer` を子に持つ。本人のみ可視（公開されない）。進行中（未完了）の Attempt は同一ユーザ×同一クイズで最大1つで、開き直すと未回答の設問から再開する。全設問への回答で完了となり、完了後は新しい Attempt として何度でも再挑戦できる。公開される挑戦数・正答率などの集計は完了した Attempt のみを数える。
_Avoid_: 回答（answer は設問単位）、トライ、チャレンジ、受験

**Answer (回答)**:
1つの設問に対する1回の提出。設問ごとに提出され、提出と同時にサーバが採点して正誤が確定する（以後変更・再提出不可）。回答の直後に正誤・正解・解説が本人へ開示される（＝[[Immediate Feedback]]）。
_Avoid_: 解答、レスポンス、提出（曖昧）

**Immediate Feedback (即時フィードバック)**:
回答1件の採点直後に、その設問の正誤・正解・解説を開示すること。クイズ全体の完了を待たない。未回答の設問の正解は開示されない。
_Avoid_: 結果表示（クイズ全体の結果と曖昧）

**Author (作者)**:
クイズを作ったユーザ。公開されるのは [[Display Name]] のみで email は非公開。
_Avoid_: クリエイター、出題者、投稿者

**Display Name (表示名)**:
作者として公開される名前。初回サインインの OAuth プロバイダから取得し、本人が後で編集可能。
_Avoid_: ユーザ名、ニックネーム、ハンドル

**Favorite (お気に入り)**:
ユーザが公開 [[Quiz]] を「あとで挑戦/復習したい」と手動で登録する、本人だけの私的コレクション。**クイズ単位**（user × quiz）。[[Attempt]] 画面のトグルで登録/解除でき、挑戦するクイズの選択元の1つになる。SRS の自動スケジュール（review_state・**設問単位**・Phase 3）とは別概念＝こちらは**手動・クイズ単位**。
_Avoid_: いいね（公開の社会的シグナルで別概念）、ブックマーク、my hot（UI ラベルとして使うことはあるが、語彙としては Favorite に寄せる）

**Streak (連続記録 / ストリーク)**:
本人が学習を継続した連続日数。[[Attempt]] の回答が1件以上あった JST 暦日が連続している長さで、現在の連続＋最長を表示する。私的指標で公開されない。日境界・活動定義・タイムゾーンの詳細は [[0006-dashboard-aggregation-semantics]]。
_Avoid_: 連勝（正解の連続ではない）、ログイン日数（基準は回答の有無）
