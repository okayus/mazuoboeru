# まず覚える (mazuoboeru)

学んだことをクイズ化して反復で覚える、マルチユーザーの公開学習サービス。
クイズ本体は必ず公開、個人の学習履歴・PII は非公開。

## Language

**Quiz (クイズ)**:
1人のユーザ（作者）が作って公開する、設問の集合体。タイトル、説明、複数の [[Question]] を含む。作成と同時に自動公開される（非公開オプションは存在しない）。
_Avoid_: テスト、試験、問題集

**Session (セッション)**:
ブラウザで Web UI を使う人間ユーザの認証状態。Cookie ベース（HttpOnly + Secure + SameSite）。サーバ側で `session` テーブル管理。
_Avoid_: ログイン情報、認証クッキー

**PAT (Personal Access Token)**:
CLI または AI エージェント（Claude 等）が API を叩くための Bearer トークン。Web の設定画面でユーザが自分で発行し、平文表示は発行直後のみ。失効・スコープを持つ。クイズの量産（AI 駆動の作成）が主な用途。
_Avoid_: API キー、シークレット、アクセスキー、APIトークン（曖昧）

**Publish (公開する)**:
作者（または作者の [[PAT]] を使う委任エージェント）による **意図的・不可逆** な明示操作。`draft` → `published` の一方向で、一度公開したクイズを `draft` に戻すことはできない（[[0002-publish-flow-and-edit-rules]]）。チャネルは問わない（Web の「公開する」ボタン、または PAT 経由の明示的な publish 呼び出し）が、いずれも「明示的な1ステップ」である点は共通（暗黙の副作用としては起こらない）。
_Avoid_: リリース、出題、公表（曖昧）

**Draft (下書き)**:
作者本人のみが見える、作成途中のクイズ。タイトルや設問が未完成でも保存できる。`Publish` の対義語的状態。
_Avoid_: 非公開、プライベート（mazuoboeru には private 状態は存在しない）

**Hidden (非表示)**:
モデレータが違反コンテンツに対して付与する状態。一般ユーザからは検索/直リンク共に閲覧不可（403）。作者本人だけ自身の hidden 状態を確認できる。`Draft` とは別概念（draft は作者の意思、hidden は他者の介入）。
_Avoid_: ブロック、削除、停止（曖昧）

**Challenge (挑戦) — UI 動詞（旧エンティティ "Attempt" は引退）**:
あるクイズの全設問を1問ずつ通しで解く操作＝**クイズ単位の [[Drill]] を始めること**（UI ラベルは「挑戦する」。初見の通しを指す自然な語として残す）。**かつての『Attempt』エンティティ**——1クイズ1回ぶんをまとめる記録で、完了・未回答からの再開・1回ごとのスコア（7/10）を持っていた——**は引退した**。回答は全て [[Answer]] としてフラットに残り、「1回ぶんの括り」も「完了」も持たない（[[0013-retire-attempt-unify-answers]]）。クイズ別の集計は [[Answer]]→[[Question]]→[[Quiz]] から導ける（グルーピング行は不要）。
_Avoid_: Attempt（引退した旧エンティティ名。グルーピング／完了／再開を含意するので“概念”としては使わない）、受験、トライ。

**Answer (回答)**:
1つの設問に対する1回の提出で、本サービスが永続化する**中心的な単位**（[[Challenge]] のような上位グルーピング＝旧 Attempt は持たない）。提出と同時にサーバが採点して正誤が確定し、その**1件は不変**（後から書き換えない＝履歴の自立性）。ただし**同じ設問へ後でまた答えれば、それは別の新しい Answer 1件**になる（『再提出不可』ではなく追記＝[[Drill]] で何度でも解き直せる）。直後に正誤・正解・解説を本人へ開示（＝[[Immediate Feedback]]）。[[Drill]]（[[Review List]] 由来でもクイズ単位でも）から生まれる回答は全てこの Answer。本人のみ可視。
_Avoid_: 解答、レスポンス、提出（曖昧）、Attempt（回答をまとめる上位記録は持たない＝[[0013-retire-attempt-unify-answers]]）

**Immediate Feedback (即時フィードバック)**:
回答1件の採点直後に、その設問の正誤・正解・解説を開示すること。クイズ全体の完了を待たない。未回答の設問の正解は開示されない。
_Avoid_: 結果表示（クイズ全体の結果と曖昧）

**Server-Side Grading (サーバー採点)**:
[[Answer]] の正誤・スコア・正答率集計を**サーバが唯一の権威**として確定すること（純粋関数 `worker/domain/grading.ts` の strict 判定＋境界）。根拠は **(1) 正誤の単一の真実・(2) [[Immediate Feedback]]（正解/解説を回答後にのみ返す）・(3) 読みモデルの分離（公開射影 `publicQuizJson` は `is_correct`/解説を含まない）**。ランキング/公開スコアが無いので**競争的 anti-cheat ではない**（挑戦者が正解を先に見ても自分の学習を損なうだけで脅威ではない＝[[0010-server-side-grading-rationale]]）。
_Avoid_: カンニング防止／不正防止（競争的 anti-cheat を含意するが本アプリにランキングは無い）、クライアント採点（正誤はサーバ権威）。

**Short Answer (短答 / 一問一答 / short)**:
[[Question]] の一形式で、選択肢から選ぶ代わりに**答えを自由入力でタイプ**する（例「プロセスが属する各 namespace へのポインタをまとめた構造体は？」→「nsproxy」）。採点はサーバー権威（[[Server-Side Grading]]）で、入力を [[Answer Normalization]] したうえで作者の [[Accepted Answer]] 集合と一致比較する。文中の空所を埋める穴埋め（cloze）は**別形式**（空所が複数あり得る）で本 MVP の対象外（短答を「空所1個の特殊形」として後付け拡張する想定＝[[0012-short-answer-grading-normalization]]）。
_Avoid_: 記述式（長文を採点する含意）、穴埋め／cloze（空所を埋める別形式）、自由回答（採点不能な含意）。

**Accepted Answer (許容解)**:
[[Short Answer]] 設問について、作者が与える「正解とみなす答えの集合」。[[Answer Normalization]] 後にいずれか1つと一致すれば正解。先頭を**正準解**とし [[Immediate Feedback]] で「正解」として表示する。かな表記・シノニム・記号のあり無しといった**意味的な揺れはここに列挙**する（正規化では畳まない＝[[0012-short-answer-grading-normalization]]）。
_Avoid_: 正解（1つに聞こえるが集合）、別解（正準解を除いた残りだけを指すが、許容解は正準解を含む全体）。

**Answer Normalization (解答正規化)**:
[[Accepted Answer]] と挑戦者の入力を比較する前に、**両辺へ同じく**かける機械的な文字の畳み込み。全角/半角・大文字小文字・前後と連続の空白といった**機械的な揺れだけ**を同一視し、それ以外（かな・シノニム等の意味的な揺れ）は畳まない＝それらは [[Accepted Answer]] に列挙して表現する。あいまい一致（編集距離・部分一致）は行わない（正規化後は完全一致）。詳細は [[0012-short-answer-grading-normalization]]。
_Avoid_: ファジー照合／あいまい一致（編集距離は使わない）、表記ゆれ吸収（機械的な揺れのみ。意味的変種は許容解側）。

**Author (作者)**:
クイズを作ったユーザ。公開されるのは [[Display Name]] のみで email は非公開。
_Avoid_: クリエイター、出題者、投稿者

**Display Name (表示名)**:
作者として公開される名前。初回サインインの OAuth プロバイダから取得し、本人が後で編集可能。
_Avoid_: ユーザ名、ニックネーム、ハンドル

**Review List (復習リスト)**:
ユーザが「まだ覚えていない・繰り返し復習したい」[[Question]] を手動で集める、本人だけの私的なプール。**設問単位**（user × question）。登録はクイズ単位の一括でもよい（そのクイズの設問をまとめて入れる）が、卒業（覚えたとして外す）は**設問単位**。[[Drill]]（[[Review List]] 由来・クイズ単位いずれも）の最中にトグルで出し入れする。「覚えた／まだ」は専用の自己評価ではなく、**このリストへの残留／除外そのもの**で表す（まぐれ正答でも外さなければ残る）。再出題の**間隔（いつ出すか）はこのリストに持たせず**、将来の通知（リマインド）に委ねる＝在庫スケジューラ（SM-2）は持たない（[[0008-review-list-manual-pool]]）。
_Avoid_: Favorite / お気に入り（クイズ単位だった旧概念。設問単位の本概念へ置き換えた）、SRS / 自動スケジュール（本リストは手動・間隔を持たない）、いいね（公開の社会的シグナルで別概念）。UI ラベルは "my hot list" を用いる（語彙は Review List に寄せる。命名は暫定で改名可）。

**Drill (ドリル)**:
**プールの設問を1問ずつ解いて定着を図る**、本人だけの学習行為。プールは2種類——[[Review List]]、または**ひとつの [[Quiz]] 全体**（=クイズ単位ドリル。UI で「挑戦」と呼ぶ初見の通しもこれ＝[[Challenge]]）。サーバー採点のうえ即時フィードバック（正誤・正解・解説＝[[Immediate Feedback]]）を返す。[[Review List]] 由来なら直後に「覚えた（外す）／まだ（残す）」を選べる。生まれた回答は [[Answer]] として履歴に残り、ダッシュボードの各指標（[[Streak]]・活動量・正答率＝全体／タグ別／**クイズ別**／設問別）へ一律算入する（[[0006-dashboard-aggregation-semantics]]）。**進行状態を持たず再開もしない**——中断したら次はプールを頭からやり直す。回答は1問ごとに [[Answer]] として記録済みなので、中断で失われるのは画面上の進行だけ。設問の提示順は（クイズ単位でも）シャッフルできる。
_Avoid_: 復習スケジュール（間隔は持たない＝[[0008-review-list-manual-pool]]）、Attempt（引退した旧エンティティ＝[[0013-retire-attempt-unify-answers]]）。

**Streak (連続記録 / ストリーク)**:
本人が学習を継続した連続日数。[[Answer]]（＝[[Drill]] による回答）が1件以上あった JST 暦日が連続している長さで、現在の連続＋最長を表示する。私的指標で公開されない。日境界・活動定義・タイムゾーンの詳細は [[0006-dashboard-aggregation-semantics]]。
_Avoid_: 連勝（正解の連続ではない）、ログイン日数（基準は回答の有無）

**Tag (タグ)**:
公開 [[Quiz]] に付く、クイズ単位の公開メタデータのラベル。作者が自由に作成・付与する UGC（フラット）。同じ語は1つに統合される（識別は正規化キー）。検索・絞り込み・ダッシュボードの軸。
_Avoid_: カテゴリ（別概念は作らない＝[[Tag Taxonomy]] の上位タグがその役割を兼ねる）、ジャンル

**Tag Taxonomy (上位・下位 / subsumption)**:
[[Tag]] 同士の有向「上位/下位（is-a・包含）」関係。「JavaScript は programming の下位」のように結び、全体は**グラフ（DAG。1タグが複数の上位を持てる多親）**を成す。運用者（moderator/admin）が curate する共有語彙で、一般ユーザは閲覧のみ。詳細は [[0007-tag-subsumption-taxonomy]]。
_Avoid_: 親子（木＝単一親を想起させるが実体は多親 DAG）、カテゴリ階層、関連（無向の汎用関連は別物で現状作らない）

**Effective Tags (実効タグ)**:
あるクイズに対し、Authored タグ（作者が実際に付けたタグ）に [[Tag Taxonomy]] の**上位すべて（DAG の上位閉包）**を加えたもの。絞り込み・ロールアップ・タグ別の集計（[[0006-dashboard-aggregation-semantics]]）は実効タグで行う。authored は保存され、実効は導出される（保存しない）。
_Avoid_: 実効タグを保存済みデータと混同する（実効は導出値）
