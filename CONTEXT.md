# まず覚える (mazuoboeru)

学んだことをクイズ化して反復で覚える、マルチユーザーの公開学習サービス。
クイズ本体は必ず公開、個人の学習履歴・PII は非公開。

## Language

**Quiz (クイズ)**:
1人のユーザ（作者）が作って公開する、設問の集合体。タイトル、説明、複数の [[Question]] を含む。作成と同時に自動公開される（非公開オプションは存在しない）。
_Avoid_: テスト、試験、問題集

**Session (セッション)**:
ブラウザで Web UI を使う人間ユーザの認証状態。Cookie ベース（HttpOnly + Secure + SameSite）。サーバ側で `session` テーブル管理。
_Avoid_: ログイン情報、認証クッキー

**PAT (Personal Access Token)**:
CLI または AI エージェント（Claude 等）が API を叩くための Bearer トークン。Web の設定画面でユーザが自分で発行し、平文表示は発行直後のみ。失効・スコープを持つ。クイズの量産（AI 駆動の作成）が主な用途。
_Avoid_: API キー、シークレット、アクセスキー、APIトークン（曖昧）

**Publish (公開する)**:
作者（または作者の [[PAT]] を使う委任エージェント）による **意図的・不可逆** な明示操作。`draft` → `published` の一方向で、一度公開したクイズを `draft` に戻すことはできない（[[0002-publish-flow-and-edit-rules]]）。チャネルは問わない（Web の「公開する」ボタン、または PAT 経由の明示的な publish 呼び出し）が、いずれも「明示的な1ステップ」である点は共通（暗黙の副作用としては起こらない）。
_Avoid_: リリース、出題、公表（曖昧）

**Draft (下書き)**:
作者本人のみが見える、作成途中のクイズ。タイトルや設問が未完成でも保存できる。`Publish` の対義語的状態。
_Avoid_: 非公開、プライベート（mazuoboeru には private 状態は存在しない）

**Hidden (非表示)**:
モデレータが違反コンテンツに対して付与する状態。一般ユーザからは検索/直リンク共に閲覧不可（403）。作者本人だけ自身の hidden 状態を確認できる。`Draft` とは別概念（draft は作者の意思、hidden は他者の介入）。
_Avoid_: ブロック、削除、停止（曖昧）

**Attempt (挑戦)**:
1人のユーザが1つのクイズに挑む1セッション分の記録。`attempt_answer` を子に持つ。本人のみ可視（公開されない）。進行中（未完了）の Attempt は同一ユーザ×同一クイズで最大1つで、開き直すと未回答の設問から再開する。全設問への回答で完了となり、完了後は新しい Attempt として何度でも再挑戦できる。公開される挑戦数・正答率などの集計は完了した Attempt のみを数える。
_Avoid_: 回答（answer は設問単位）、トライ、チャレンジ、受験

**Answer (回答)**:
1つの設問に対する1回の提出。設問ごとに提出され、提出と同時にサーバが採点して正誤が確定する（以後変更・再提出不可）。回答の直後に正誤・正解・解説が本人へ開示される（＝[[Immediate Feedback]]）。
_Avoid_: 解答、レスポンス、提出（曖昧）

**Immediate Feedback (即時フィードバック)**:
回答1件の採点直後に、その設問の正誤・正解・解説を開示すること。クイズ全体の完了を待たない。未回答の設問の正解は開示されない。
_Avoid_: 結果表示（クイズ全体の結果と曖昧）

**Server-Side Grading (サーバー採点)**:
[[Answer]] の正誤・スコア・正答率集計を**サーバが唯一の権威**として確定すること（純粋関数 `worker/domain/grading.ts` の strict 判定＋境界）。根拠は **(1) 正誤の単一の真実・(2) [[Immediate Feedback]]（正解/解説を回答後にのみ返す）・(3) 読みモデルの分離（公開射影 `publicQuizJson` は `is_correct`/解説を含まない）**。ランキング/公開スコアが無いので**競争的 anti-cheat ではない**（挑戦者が正解を先に見ても自分の学習を損なうだけで脅威ではない＝[[0010-server-side-grading-rationale]]）。
_Avoid_: カンニング防止／不正防止（競争的 anti-cheat を含意するが本アプリにランキングは無い）、クライアント採点（正誤はサーバ権威）。

**Author (作者)**:
クイズを作ったユーザ。公開されるのは [[Display Name]] のみで email は非公開。
_Avoid_: クリエイター、出題者、投稿者

**Display Name (表示名)**:
作者として公開される名前。初回サインインの OAuth プロバイダから取得し、本人が後で編集可能。
_Avoid_: ユーザ名、ニックネーム、ハンドル

**Review List (復習リスト)**:
ユーザが「まだ覚えていない・繰り返し復習したい」[[Question]] を手動で集める、本人だけの私的なプール。**設問単位**（user × question）。登録はクイズ単位の一括でもよい（そのクイズの設問をまとめて入れる）が、卒業（覚えたとして外す）は**設問単位**。[[Attempt]] や [[Drill]] の最中にトグルで出し入れする。「覚えた／まだ」は専用の自己評価ではなく、**このリストへの残留／除外そのもの**で表す（まぐれ正答でも外さなければ残る）。再出題の**間隔（いつ出すか）はこのリストに持たせず**、将来の通知（リマインド）に委ねる＝在庫スケジューラ（SM-2）は持たない（[[0008-review-list-manual-pool]]）。
_Avoid_: Favorite / お気に入り（クイズ単位だった旧概念。設問単位の本概念へ置き換えた）、SRS / 自動スケジュール（本リストは手動・間隔を持たない）、いいね（公開の社会的シグナルで別概念）。UI ラベルは "my hot list" を用いる（語彙は Review List に寄せる。命名は暫定で改名可）。

**Drill (ドリル)**:
[[Review List]] の設問を1問ずつ解き直して定着を図る、本人だけの学習行為。サーバー採点のうえ即時フィードバック（正誤・正解・解説＝[[Immediate Feedback]]）を返し、直後に「覚えた（リストから外す）／まだ（残す）」を選ぶ。ドリルも「その日学習した」活動として数え、[[Streak]] に算入する（[[0006-dashboard-aggregation-semantics]]）。クイズを通しで挑む [[Attempt]] とは別物（プールから設問をつまむ・1クイズ単位ではない・完了概念がない）。
_Avoid_: [[Attempt]]（1クイズ通し・完了あり）、復習スケジュール（間隔は持たない）。

**Streak (連続記録 / ストリーク)**:
本人が学習を継続した連続日数。[[Attempt]] の回答または [[Drill]] が1件以上あった JST 暦日が連続している長さで、現在の連続＋最長を表示する。私的指標で公開されない。日境界・活動定義・タイムゾーンの詳細は [[0006-dashboard-aggregation-semantics]]。
_Avoid_: 連勝（正解の連続ではない）、ログイン日数（基準は回答の有無）

**Tag (タグ)**:
公開 [[Quiz]] に付く、クイズ単位の公開メタデータのラベル。作者が自由に作成・付与する UGC（フラット）。同じ語は1つに統合される（識別は正規化キー）。検索・絞り込み・ダッシュボードの軸。
_Avoid_: カテゴリ（別概念は作らない＝[[Tag Taxonomy]] の上位タグがその役割を兼ねる）、ジャンル

**Tag Taxonomy (上位・下位 / subsumption)**:
[[Tag]] 同士の有向「上位/下位（is-a・包含）」関係。「JavaScript は programming の下位」のように結び、全体は**グラフ（DAG。1タグが複数の上位を持てる多親）**を成す。運用者（moderator/admin）が curate する共有語彙で、一般ユーザは閲覧のみ。詳細は [[0007-tag-subsumption-taxonomy]]。
_Avoid_: 親子（木＝単一親を想起させるが実体は多親 DAG）、カテゴリ階層、関連（無向の汎用関連は別物で現状作らない）

**Effective Tags (実効タグ)**:
あるクイズに対し、Authored タグ（作者が実際に付けたタグ）に [[Tag Taxonomy]] の**上位すべて（DAG の上位閉包）**を加えたもの。絞り込み・ロールアップ・タグ別の集計（[[0006-dashboard-aggregation-semantics]]）は実効タグで行う。authored は保存され、実効は導出される（保存しない）。
_Avoid_: 実効タグを保存済みデータと混同する（実効は導出値）
