# 機能仕様

マルチユーザーの公開サービス。中心は「クイズを作る」「クイズに挑戦する」「復習で覚える」。

## 1. アカウント

- マルチユーザー。サインアップ／ログイン。
- 表示名（公開）とメール等（非公開）を分離。
- 認証は **OAuth (Google + GitHub) + PAT**（[ADR-0001](adr/0001-auth-via-oauth-and-pat.md)）。
  - Web (人間): Google または GitHub のいずれか。同一 verified email は自動リンクして1ユーザに統合。
  - CLI / AI エージェント (Claude 等): Web 設定画面で **PAT (Personal Access Token)** を発行 → Bearer で API を叩く。AI 駆動のクイズ量産が主な動機。
  - Passkey は MVP 範囲外（Phase 2 で Google ログイン後の追加手段として再評価）。
- プロフィール: 表示名、作ったクイズ一覧、ざっくりした活動量。PAT 管理画面（発行・失効）を Phase 1 で同梱。

## 2. クイズ作成（必ず公開）

- クイズ = タイトル＋説明＋複数の設問。タグ付け可。
- **設問の形式**:
  - **MVP**: 多肢選択（`mcq_single` / `mcq_multi`、複数正解は **全一致で正解** の strict 採点）。
  - **Phase 2 候補**: ○×（`boolean`）、一問一答（`short`、表記ゆれの正規化が要詰め）、穴埋め（`cloze`）。
  - 設問形式は将来追加が容易な形でスキーマを設計する（`question.type` を text enum、type 別 payload は将来 JSON 等で）。
- 各設問に **解説** を付けられる（挑戦後に表示。学びの核）。
- Markdown 対応（コード・リンク程度）。**表示時はサニタイズ必須**（[security.md](security.md)）。
- 公開フロー（[ADR-0002](adr/0002-publish-flow-and-edit-rules.md)）:
  - 作成中は `draft`（本人のみ可視）。
  - 作者が「公開する」ボタンを押すと `published` に **不可逆遷移**。
  - 公開後の **軽微編集**（タイトル・説明・設問文・解説）は自由。**重大編集**（正解・選択肢の追加削除）は UI で警告のみ、過去の `attempt_answer.is_correct` は再計算しない。
  - 削除は **ソフトデリート** (`deleted_at`)。タイムライン・検索・直リンクから消える。`attempt` 履歴は残る。
- `hidden` は通報→違反確定時のモデレータ操作専用。作者本人の self-hide はソフト削除で代替。

## 3. 挑戦・採点

- 自分／他人のクイズに挑戦。1問ずつ or まとめて。
- **採点はサーバー側**で行う（正解をクライアントに事前送信しない＝不正防止）。
- 即時フィードバック: 正誤＋解説表示。
- 結果を記録（挑戦＝attempt、各設問の回答＝answer）。スコア・所要時間。
- 同じクイズへの再挑戦可（成績は履歴として残る）。

## 4. 復習（SRS：間隔反復）

- 間違えた設問・自信のなかった設問を **間隔反復** で再出題し、定着を支援。
- 「今日の復習」キュー。覚えた/まだ、の自己評価で次回間隔を調整（SM-2 系の簡易版から）。
- 復習スケジュールは本人のみ（非公開）。

## 5. 発見（公開プールを活かす）

- 検索（クイズタイトル・設問・タグの全文）。
- タグ／カテゴリでの絞り込み。
- 新着 / 人気（挑戦数・お気に入り数）/ ランキング。
- 作者ページから辿る。

## 6. 学習ダッシュボード（本人のみ）

- 自分の正答率・学習履歴・連続記録（ストリーク）。
- タグ別の習熟度。
- 作ったクイズの反響（挑戦数・平均正答率など集計）。

## 7. 軽いソーシャル

- 作者表示、**お気に入り / いいね**、挑戦数の表示。
- コメント機能は **要検討**（モデレーション負荷が大きいので初期は見送り候補）。

## 8. モデレーション（公開サービスの必須機能）

- 不適切クイズの **通報**（クイズ/設問/ユーザの3粒度、選択肢理由＋自由記述、最大500文字、1ユーザ10件/日のレート制限）。
- 管理者によるクイズの **`hidden`**／削除、ユーザの制限（`user.status='suspended'`）。
- スパム・大量自動投稿の抑制（レート制限、`cloudflare-workers-bot-scan-defense`）。
- 著作権侵害・不適切表現への対応フロー（運用ドキュメント込み、プロバイダ責任制限法 / DSA / DMCA の最低線を満たす）。

## 機能の優先度

| 優先度 | 機能 |
| --- | --- |
| **MVP 必須** | アカウント/認証 (OAuth Google+GitHub)、**PAT 発行・管理**、クイズ作成 (`mcq_single`+`mcq_multi` ＋解説)、明示公開フロー、挑戦＆サーバー採点、公開タイムライン、**通報チャネル**（ボタン + endpoint + テーブル + rate limit） |
| 高 | 検索・タグ・発見、学習ダッシュボード、お気に入り、他の設問形式（`boolean` → `short` → `cloze`）、通報の Discord/メール通知 |
| 中 | SRS 復習、ランキング、作者ページ、モデレーション管理画面、Cloudflare Access で `/admin` ゲート、Passkey 追加導線 |
| 低 | コメント、編集履歴 (`quiz_revision`)、AI 出題補助、PWA/オフライン、`apps/cli` の npm 配信、ハード削除・GDPR エクスポート |

段階分けは [roadmap.md](roadmap.md) を参照。
