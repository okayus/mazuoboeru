# セキュリティ・プライバシー設計

mazuoboeru は **マルチユーザーの公開サービス**。kokemusu（単一ユーザー私的）と脅威モデルが大きく異なる。
クイズ本体は公開前提なので「本文の機密性」は争点でないが、代わりに **UGC の安全な表示・他者からの不正・アカウント保護・モデレーション** が中心になる。

## 脅威モデル

| 脅威 | 対策の方向性 |
| --- | --- |
| 他人のクイズ（UGC）経由の **XSS** | react-markdown で AST 描画（生 HTML を描画しない）＋ rehype-sanitize、CSP。MVP は画像を描画しない |
| アカウント乗っ取り | 強い認証（OAuth/パスキー）、セッション安全、レート制限 |
| ~~不正回答・カンニング（正解の盗み見）~~ → **現状は脅威対象外** | ランキング/公開スコアが無く学習は内面的＝競争的カンニングは脅威でない。サーバー採点は維持するが根拠は別（単一真実・即時FB・読みモデル分離＝[ADR-0010](adr/0010-server-side-grading-rationale.md)）。将来ランキング導入時に再掲 |
| スパム / 大量自動投稿 / bot スキャン | レート制限、投稿スロットリング、CAPTCHA 検討 |
| 不適切・違法コンテンツの公開 | 通報＋モデレーション、削除フロー |
| 個人情報（メール等）の漏洩 | PII を公開面に出さない、最小収集、保存時暗号化（インフラ層） |
| なりすまし（OAuth ID 偽装） | verified email 限定の自動リンク（[ADR-0001](adr/0001-auth-via-oauth-and-pat.md)）、入力検証（Zod）。**スコア改ざん**は公開スコア/ランキングが無く現状は対象外（[ADR-0010](adr/0010-server-side-grading-rationale.md)） |
| CSRF | SameSite=Lax ＋ 状態変更メソッドの Origin 検証（決定済み。PAT/Bearer 経路は ambient credential がなく対象外） |
| 依存の脆弱性 | 依存最小化、定期監査・更新 |

## 認証（マルチユーザー）── 要決定

候補（[ADR-0001](adr/0001-auth-via-oauth-and-pat.md) で確定済み）:

- **OAuth ソーシャルログイン（Google 等）**: 学習サービスとの相性が良く、導入が手軽。パスワード管理不要。
- **パスキー / WebAuthn**: フィッシング耐性。マルチユーザーでも可。kokemusu で採用予定の方式を再利用できる。
- **併用**: ソーシャル＋パスキーの両対応。

共通の必須事項:
- セッションはサーバー側管理。Cookie は `HttpOnly` `Secure` `SameSite`。
- ログイン試行のレート制限、ログイン履歴。
- パスワードを使う場合は Argon2id。
- ⚠️ パスキーを使うなら **RP_ID（本番ドメイン）を最初に固定**（後で変えると登録済みパスキー無効）。
  kokemusu と同様 `<project>.workers.dev` を day 1 から固定する想定。

## UGC（ユーザー生成コンテンツ）の安全な表示 ── 最重要

他人が書いたクイズ本文・解説を、別のユーザーのブラウザに表示する。ここが最大の攻撃面。

- **描画は `react-markdown`（決定済み・Phase 1 グリル）**。Markdown を AST 経由で React 要素へ描画し、HTML 文字列も `dangerouslySetInnerHTML` も使わない（`rehype-raw` を入れないので生 HTML は描画されない）。`rehype-sanitize`（strict schema）を多層防御に重ね、`on*` 属性 / `javascript:` href / `<script>` を除去。
- **拡張は plugin / components で差し替え可能に**: 描画は単一の共有コンポーネント（例 `<QuizMarkdown>`）へ集約し、remark/rehype プラグイン列と要素マップを一点管理。Markdown **原文を保存し描画時にレンダリング**するため、将来 **mermaid（```mermaid フェンス → `code` 描画の差し替え）や数式（KaTeX）** を保存形式の移行なしで追加できる。ただし mermaid は SVG 出力＝新たな XSS 面（SVG サニタイズ＋ `securityLevel:'strict'` が要る）なので MVP では描画せず Phase 2。
- **CSP**（インラインスクリプト禁止、`img-src 'self'` まで限定）で多層防御。MVP は画像（`![]()`）を描画せずリンク表示にし、外部リソース読み込み＝閲覧者の IP/UA 漏洩・トラッキング・混在コンテンツ面をゼロにする（画像許可は Phase 2 で再検討）。
- 文字数・サイズ制限、入力検証（Zod）でペイロードを制限。

## 採点（サーバー権威・[ADR-0010](adr/0010-server-side-grading-rationale.md)）

採点をサーバー側に置く根拠は **競争的 anti-cheat ではない**（ランキング/公開スコアが無く学習は内面的＝挑戦者が正解を先に見ても自分の学習を損なうだけで他者にも集計にも害がない）。次の3つが根拠:

- **正誤の単一の真実**: 採点・スコア・正答率の集計はサーバーが唯一の権威。クライアントから来たスコアを信用しないのは改ざん防御ではなく整合性のため。
- **即時フィードバック**: 正解・解説は回答1件の採点後にのみ返す（[CONTEXT.md](../CONTEXT.md) Immediate Feedback）。クイズ全体の完了を待たない。
- **読みモデルの分離**: 公開射影（`worker/presenters/quiz.ts` の `publicQuizJson`）は `is_correct`・解説を含まない＝「問題の読みモデル」と「採点後の回答レスポンス」を分ける設計の綺麗さ（セキュリティ境界ではない）。
- 連投・総当たり的な回答送信のレート制限は別途維持（理由は DoS/コスト面であって anti-cheat ではない）。

## モデレーション

- 通報導線（クイズ/設問単位）。
- 管理者ロールで非表示・削除・ユーザー制限。
- 自動フィルタ（NG ワード・スパム検知）は段階的に。
- 監査ログ（誰が何を消したか）。

## 通信・Web セキュリティ

- HTTPS 必須・HSTS。
- セキュリティヘッダ一式（CSP / `X-Content-Type-Options` / `Referrer-Policy` / frame-ancestors 等）。
- CSRF 対策（決定済み・Phase 1 グリル）: SameSite=Lax を第一防御、状態変更メソッド（POST/PATCH/DELETE）で Origin ヘッダを ORIGIN 設定値と照合する middleware を第二防御。同期トークンは不採用（必要になれば後付け可能＝可逆）。
- 認証・投稿などの非公開でない入口に **レート制限**（`cloudflare-workers-bot-scan-defense` スキル）。

## プライバシー

- 公開: クイズ本体・作者表示名・クイズ単位の集計。
- 非公開（本人のみ）: 挑戦履歴・成績・正答率・復習リスト/ドリルの記録・メール等の PII。
- データエクスポート/削除（アカウント削除時の扱い）を将来用意。
