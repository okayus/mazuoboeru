# コンセプト

## 名前の由来 ── 「まず覚える」

何かを学ぼうとするとき、理解の前にまず暗記が要る場面は多い（単語、定義、用語、手順、規則…）。
「まず覚える」は、その最初のハードルを **クイズ化と反復** で越えるためのアプリ。

そして、最も効く覚え方は **自分で問題を作ること**。
出題するには、何が要点で・何が紛らわしいかを自分で判断しなければならない。
クイズを作る行為そのものが、最も能動的な学習になる（"教えることは二度学ぶこと"）。

## コア体験

1. **学ぶ → 問題にする**: 学習した内容を、設問＋答え＋（できれば）解説の形にして登録する。
2. **必ず公開される**: 作った問題は自動的に公開され、他の人も挑戦できる教材になる。
3. **挑戦する**: 自分や他人のクイズに挑戦し、即時に採点・フィードバックを受ける。
4. **反復で定着**: 覚えきれない設問は復習リスト（"my hot list"）に集め、繰り返し解いて「まず覚える」を後押しする（リマインドは将来の通知で）。

## 「クイズは必ず公開」という設計制約

これは意図的な中心思想であり、非公開オプションは作らない。理由:

- **知識を共有財にする**: 一人の学びを、同じことを学ぶ全員の教材に変える。車輪の再発明を減らす。
- **作る側の質を上げる**: 公開前提だと、自然と「人に見せられる問題」を作ろうとする緊張感が働く。
- **発見の母数を増やす**: 公開クイズが増えるほど、挑戦できる教材プールが豊かになる正のループ。
- **設計をシンプルに保つ**: 公開/非公開の出し分け・権限分岐がなく、表示ロジックが単純になる。

> ⚠️ トレードオフ: 公開前提なので **個人情報・私的メモはクイズに書かない** という前提が要る。
> 「私的な記録」は kokemusu の領分。mazuoboeru は「共有して良い知識」を扱う。この境界は UI とガイドで明示する。
> （※「必ず公開」は後戻りしにくい根幹の決定なので ADR 化候補。`/grill-with-docs` で詰める。）

## 公開するもの / しないもの

| 公開される | 公開されない（本人のみ） |
| --- | --- |
| クイズ本体（設問・選択肢・正解・解説） | 自分の挑戦履歴・成績・正答率 |
| 作者の表示名 | メールアドレス等の個人情報 |
| クイズの挑戦回数・平均正答率などの集計 | 復習リスト（"my hot list"）・ドリルの記録 |

## 設計上の価値観

- **作る摩擦を下げる**: クイズ作成を面倒にしない。最小フォーム＋後から育てられる編集。
- **能動 > 受動**: 読むだけでなく「答える」「作る」を中心に置く。
- **共有のループ**: 良問が見つかる → 挑戦が増える → 作る人が増える、を回す。
- **公正な採点**: 正誤判定はサーバー側。スコアの不正やカンニングを構造的に防ぐ。

## やらないこと（初期スコープ外）

- 非公開クイズ / 限定共有（「必ず公開」の思想に反する）。
- 重いソーシャル（DM、フォロー中心のタイムライン等）。学習が主、ソーシャルは従。
- AI 自動出題（将来の補助機能としては検討余地あり。初期は人が作る）。
- 課金・有料コンテンツ（初期は無料・共有が前提）。
