# 学習ダッシュボードの集計セマンティクス（私的・回答単位・活動量ベース／ストリークは JST 日次）

## Context

Phase 2 の学習ダッシュボード（振り返り）は、既存の [[Attempt]] / `attempt_answer` から **本人だけの**指標（全体正答率・タグ別正答率・設問別正答率・ストリーク）を出す。新テーブルは増やさず読みで完結させるのが前提（タグ別だけは `tag`/`quiz_tags` に依存）。

ここで2つの定義問題がある。

1. **集計の単位**: CONTEXT.md の [[Attempt]] は「**公開**される挑戦数・正答率は**完了した Attempt のみ**を数える」と定める。だが今回の UX 要件は「回答した時点で集計され、クイズを完了せず離脱してもダッシュボードに反映される」。完了 Attempt 単位だと部分挑戦が消える。さらに [[Immediate Feedback]] により**同じ設問への2回目以降の回答はオープンブック**になり、「正答率」が定着度なのか活動量なのかで意味が変わる。
2. **ストリークの暦日境界**: タイムスタンプは epoch ms（UTC）保存。連続記録は「暦日」の数なのでタイムゾーン依存で、UTC で切ると 09:00 JST がリセット時刻になる。

## Decision

**私的ダッシュボードは公開集計と別基準を採る**（CONTEXT.md の「完了のみ・クイズ単位」は *公開* 集計の定義であり、本人だけに見える私的指標は対象外）。

- **集計単位 = 回答（`attempt_answer`）単位**。完了 [[Attempt]] を待たず、回答1件＝1行が即時に指標へ反映される。
- **活動量ベース**（全回答をカウント。再挑戦＝オープンブックの再回答も含める）。これは「定着度（mastery）」ではなく「活動と慣れ」の指標。**定着の厳密測定はこのダッシュボードの役割にせず、SRS（`review_state`・Phase 3）とタグ別習熟度に委ねる**。
- 粒度は **全体／タグ別（回答の属する [[Quiz]] の実効タグ＝authored＋上位閉包 [[0007-tag-subsumption-taxonomy]] で束ね、タグ無しは「タグなし」バケツ。各実効タグへ多重計上）／設問別（本人の通算・挑戦中に表示）**。
- **ストリーク = JST 暦日**で、その日に**回答が1件以上**（`answered_at`）あれば成立。**現在の連続日数＋最長**を表示。JST は当面**決め打ち**で、グローバルユーザー登場時に per-user タイムゾーンへ移行する。
- **公開集計（公開表示の挑戦数・正答率）は従来どおり完了 [[Attempt]] のみ・クイズ単位を維持**。私的と公開で基準が異なることを明示する。

## Considered Options

- **集計単位**: 完了 Attempt の Σscore/Σtotal ／ **回答単位（採用）** ／ 初回回答のみ（cold mastery）。→「回答即反映・部分挑戦も数える」UX と、再挑戦を素直に取り込む活動量フレーミングに最も合う回答単位を採用。cold mastery は SRS の関心として分離。
- **ストリークの日境界**: UTC ／ **JST 固定（採用）** ／ per-user tz（将来）。→ 日本在住の現ユーザーに直感的な JST。保存は UTC のままなので将来 per-user へ移れる（表示計算の変更で済む）。
- **ストリークの活動定義**: **回答1件以上（採用）** ／ 完了 Attempt 1件以上。→「今日学習した」に最も素直で、部分挑戦でも途切れない。

## Why

- UX 要件「回答したら即記録・離脱しても反映」に、`attempt_answer`（1回答1行・サーバー採点済み）がそのまま応える＝**新テーブル不要・集計クエリのみ**。完了を待つと部分挑戦が消える。
- 活動量ベースは設問別で「繰り返すほど自分の正答率が上がる」のが見え、ドリルの**動機づけ**になる。定着の厳密測定を背負わせない＝**関心の分離**（SRS）。
- JST 決め打ちは UTC のリセット時刻問題（朝の学習が前日扱い）を避ける。
- is_correct は書込後不変（[[0002-publish-flow-and-edit-rules]]）。集計は既存行を読むだけで**履歴を再計算しない**原則と整合する。

## Consequences

- **私的と公開で集計基準が異なる**（私的＝回答単位/活動量、公開＝完了/クイズ単位）。`docs/data-model.md` §集計 と実装コメントで明示する。
- 正答率は再挑戦で上振れする（オープンブック）。**定着度ではなく活動量**であることを表示文言で正直に示す（例「これまでの平均正答率」）。
- ストリークの **JST 固定**は、グローバル化時に per-user tz への移行を要する（保存は UTC epoch ms のままなので、表示計算＋ストリーク再計算の変更で済む）。後戻りコストを承知のうえ当面 JST。
- **タグ別正答率は `tag`/`quiz_tags`/`tag_edge` の実装に依存**（実効タグ＝上位閉包で束ねる＝[[0007-tag-subsumption-taxonomy]]）。全体・設問別は既存テーブルで出せる。

## 追記

- **2026-06-18 / 復習（[[Drill]]）をストリーク・活動量に算入（[[0008-review-list-manual-pool]]）**: 復習機能を SM-2 在庫スケジューラではなく手動の [[Review List]]＋[[Drill]] で実装する決定（[[0008-review-list-manual-pool]]）に伴い、本 ADR の**ストリーク／活動量の源を拡張**する。ストリークは「その JST 暦日に `attempt_answer` **または** `review_answer`（ドリルの回答）が1件以上あるか」で判定し、活動量（総回答数）にも `review_answer` を含める。理由: 設問単位 Review List の目的が「苦手設問を毎日回す」＝ドリルが主要な学習手段になり得るため、ドリルが活動として数えられないとストリークが学習実態とズレる。正答率の扱いは従来どおり**活動量フレーミング**（オープンブックの再回答も含む・定着の厳密測定はしない）を維持する。`review_answer` は `attempt`/`attempt_answer` には乗せない（[[Attempt]]＝1クイズ通しの意味を保つ）。

- **2026-06-19 / Slice 2 grill: ドリル回答を全ダッシュボード指標へ一律算入（(2) uniform）**: 上記 2026-06-18 は `review_answer` を**ストリーク／活動量のみ**の源に挙げていたが、Slice 2 の grill で「`review_answer` は履歴の1回答＝普通の回答」と確定した。よって `review_answer` は**全指標に一律算入**する＝ストリーク・活動量に加え **全体／タグ別／設問別の正答率**にも含める（ドリルはオープンブックの再回答で、本 ADR の活動量フレーミングと無矛盾）。整合根拠: 活動量にだけ足して正答率の母数に足さないと、総回答数と正答率の分母が乖離して読み手を惑わせる。実装: ダッシュボードの純関数（`computeStreak`／`bundleTagAccuracy`／全体・設問別集計）は**不変**のまま、`review_answer` を `question`→`quiz` で join して `{isCorrect, answeredAt, quizId}` の回答ファクトへ写像し、`attempt_answer` 由来のファクトに**合流（concat）**してから同じ関数群へ通す。`review_answer` に `quiz_id` 列は持たせない（読み時 join で導出）。`gradeQuestion`（[[0010-server-side-grading-rationale]]）で採点した [[Drill]] の正誤がそのまま集計に入る。

- **2026-06-24 / [[Attempt]] 引退に伴う supersede（[[0013-retire-attempt-unify-answers]]）**: 本 ADR が前提していた [[Attempt]]／`attempt_answer`／`review_answer` の三者は、ADR-0013 で**単一のフラット `answer(id, user_id, question_id, is_correct, answered_at)` に統合**される（[[Attempt]] エンティティは引退・「挑戦」はクイズ単位 [[Drill]] の UI 動詞として残る）。これに伴い本 ADR の次の2点は**無効化＝ADR-0013 が上書きする**: (1) 上記 Decision／Consequences の「**公開集計は完了 [[Attempt]] のみ・クイズ単位**」→ 完了概念が消えるため、公開のクイズ別統計を作る場合は `answer` を question→quiz で数える（"完了" では絞らない）。(2) 2026-06-18 追記末尾「`review_answer` は `attempt`/`attempt_answer` には乗せない（[[Attempt]]＝1クイズ通しの意味を保つ）」→ まさにその分離を解消する。**私的ダッシュボードの集計セマンティクス本体（回答単位・活動量フレーム・JST ストリーク・全指標一律算入）は不変**で、集計の source が `attempt_answer`∪`review_answer` から単一 `answer` に変わるだけ（純関数群 `computeStreak`／`bundleTagAccuracy`／全体・設問別集計と facts 写像は維持）。実装は ADR-0013 のスコープ（別セッション）。
